###Réponses Questions

1. Quelle est la différence entre `git add` et `git commit` ? 
 Git add permet de sélectionner enregistrer les modifications qui seront ensuite validé par le Git commit.

2. Pourquoi utilise-t-on des branches plutôt que de modifier directement `main` ? 
- Il faut toujours gardé la branche principale propre et fonctionnelle en permanence; 
- Permet d'isoler le travail pour rajouter des fonctionnalités, modification etc;
- facilite le travail en collaboration 

3. Que se passe-t-il si tu fais `git push` sans avoir fait `git pull` au préalable ?
Il ne se passera rien si il y a eu des modifications enregistrées au préalable car git pull est nécessaire pour récupérer les modifications. 

4. À quoi sert la zone de staging (`git add`) ? Pourquoi ne pas commiter directement ? 
C'est la zone ou terrain de "préparation" qui permet de contrôler, trier, modifier, garder ce dont on a besoin. Si on commitait directement on pourrait inclure des erreurs.
